# Production Agentic Engineering

Architectural retrospectives on building and running a production agentic AI
engineering organization at a major financial institution — a 25-person team,
AWS lakehouse on Apache Iceberg, agentic delivery under a governance-first model.

Written from practice, not theory. The metrics are mechanism-anchored and the
failures are reported alongside the results.

## The series

1. [Governance Before Velocity: What Actually Made Agentic Engineering Work in Production](./01-governance-before-velocity.md)
   — the three-layer control model: data contracts, self-validating pipelines, enforced rules files.
2. [The Highest-ROI Agentic Use Case Nobody Talks About: Schema Mapping](./02-agentic-schema-mapping.md)
   — proposal problems over structured inputs with deterministic acceptance tests.
3. [Your Engineers Aren't Afraid of AI. They're Afraid of Being Reviewers.](./03-engineers-as-reviewers.md)
   — the identity transition from author to accountable reviewer, and how it was led.
4. *Forthcoming* — the metrics that survive scrutiny, and the ones that don't.
